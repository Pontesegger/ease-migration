Code Completion

Code completion in Eclipse is triggered by hitting ctrl-space in an editor. It provides suggestions, how to continue a line of code and can save you lots of typing. You may safely trigger code completion anytime to improve your productivity.

JavaScript

The JSDT editor comes with built-in completion support for your used variables and functions. Additionally EASE comes with addtional suggestions for methods of loaded modules. Therefore it parses the script code to detect loaded modules. The parser is quite limited and simply scans for loadModule() lines. It does not detect conditional loads or module names stored in variables or similar complex code constructs. Code completion is also available for several parameters (typically of string type), eg you may trigger code completion after typing loadModule(" to get a list of available modules.

To activate code completion for java types, you need to tell the editor which java object you are working on: // @type java.io.File var logFile = ... This will treat logFile as java.io.File and furtheron displays completion proposals for that variable.

Shell

Ctrl-space works in the Script Shell input box, too. Here we have live information for all variables and can automatically denote their type. This makes code completions much more powerful.
